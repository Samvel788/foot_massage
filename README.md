# Foot Massage Urgench — лендинг

Статический сайт-лендинг массажного салона на чистом HTML/CSS/JS (ES-модули),
без сборки и npm-зависимостей. Разобран из исходного «download»-бандла
(самораспаковывающийся артефакт) в нормальную модульную структуру.

## Запуск

JS подключён как `<script type="module">`, а ES-модули **не работают через
`file://`** (двойной клик по `index.html`) — нужен локальный сервер:

```bash
# любой из вариантов из корня проекта:
npx serve            # или:  npx http-server
python -m http.server
```

Затем открой `http://localhost:<порт>`. В VS Code удобно расширение **Live
Server** (правый клик по `index.html` → «Open with Live Server»).

## Структура

```text
index.html                 — разметка + SEO-мета + JSON-LD
assets/
  css/styles.css           — все стили лендинга
  js/
    main.js                — entry-point: импортирует и инициализирует модули
    i18n-data.js           — словарь переводов ru/uz/en (чистые данные)
    i18n.js                — применение языка + переключатель
    scroll.js              — общий rAF-троттлинг скролла (1 слушатель)
    nav.js                 — тень хедера + scroll-spy мобильной навигации
    reveal.js              — reveal-анимации (IntersectionObserver)
    gallery.js             — слайдер галереи (стрелки, точки, свайп)
    modal.js               — видео-модал (focus-trap, Esc)
    map.js                 — ленивая карта Leaflet (конфиг координат здесь)
    image-slot.js          — drag-and-drop заливка картинок (для владельца)
  img/
    foot_massage.png       — логотип (favicon / OG / соцсети)
    placeholder.svg        — заглушка для незаполненных фото
```

Внешние зависимости — через CDN (не зашиты в файл):
- **шрифты** Lora + Manrope — Google Fonts;
- **карта** Leaflet 1.9.4 — unpkg, грузится лениво, только когда блок карты
  появляется во вьюпорте.

## Картинки (гибрид: SEO + удобство)

Каждое фото — это настоящий `<img>` с `alt`, `width/height` и `loading`,
поэтому индексируется поисковиками и доступно скринридерам. Поверх работает
лёгкое прогрессивное улучшение `image-slot.js`: **перетащи** файл на любое
фото → оно заменится и сохранится в браузере (`localStorage`). Диалога выбора
файла по клику нет.

Сохранение живёт только в этом браузере. **Для продакшна положи реальные файлы
и пропиши `src`** в `index.html` (слоты помечены атрибутом `data-slot`):

| `data-slot`          | где                 | что за фото               |
|----------------------|---------------------|---------------------------|
| `hero-poster`        | первый экран        | атмосфера салона / постер |
| `svc-facial`         | блок «Услуги»       | массаж лица               |
| `g1a…g1d`, `g2a…g2d` | галерея (2 сцены)   | интерьер (8 фото)         |
| `spec-1`, `spec-2`   | специалисты         | портреты мастеров         |
| `why-1`, `why-2`     | «Почему мы»         | масла / кабинет           |

## Перед запуском — заменить плейсхолдеры

1. **Контакты** — телефон `+998 90 123 45 67` и email `info@footmassage.uz`
   встречаются в `index.html` (хедер, блок «Контакты», JSON-LD).
2. **Домен** — `https://footmassage.uz/` в `<link rel="canonical">`, OG-тегах и
   JSON-LD заменить на реальный.
3. **Адрес/координаты** — адрес в ключе `find.addr.v` (`assets/js/i18n-data.js`),
   координаты `MAP` в `assets/js/map.js` и `geo` в JSON-LD (сейчас центр Ургенча).
4. **Соцсети** — ссылки `#` в подвале.
5. **Видео** — у `<video>` в `index.html` пустой источник; добавь `src` или
   `<source>` (тогда кнопка play и модалка включатся автоматически).

## Что сделано при рефакторинге (аудит)

### Архитектура

- Монолитный 720-КБ self-extracting бандл → читаемые файлы.
- JS разбит на ES-модули по одной ответственности; словарь переводов отделён
  от логики; общий скролл-диспетчер вместо нескольких слушателей.
- Drag-and-drop scaffold на 30 КБ заменён на ~2 КБ прогрессивного улучшения.

### SEO

- Open Graph + Twitter Card, canonical, theme-color, favicon/apple-touch-icon.
- JSON-LD `HealthAndBeautyBusiness` (адрес, гео, часы, телефон, цены).
- Плейсхолдеры-картинки заменены на реальные `<img>` с `alt`.

### Доступность (WCAG AA)

- Skip-link, видимый `:focus-visible` на всех интерактивах.
- Вторичный текст (`--muted`) затемнён до контраста ≥ 4.5:1.
- `alt` у всех изображений, локализуется вместе с языком (`data-i18n-alt`).
- `aria-label` у иконочных кнопок и лендмарков, `aria-pressed` у переключателя
  языка, focus-trap и возврат фокуса в видео-модалке, `Esc` закрывает.
- Декоративные SVG помечены `aria-hidden`. Учтён `prefers-reduced-motion`
  (выключаются smooth-scroll и пульсация кнопки play).

### Производительность

- Leaflet (~56 КБ) и шрифты убраны из критического пути → CDN + ленивая
  загрузка карты по `IntersectionObserver`.
- Hero-изображение: `fetchpriority="high"`; остальные — `loading="lazy"`.
- У всех `<img>` заданы `width/height` → нет сдвигов макета (CLS).
- Скролл-эффекты (хедер, scroll-spy) троттлятся через `requestAnimationFrame`;
  reveal переведён на `IntersectionObserver` (без обработчика скролла).
